# Platform Architecture

Use this area for Architecture Decision Records (ADRs) in generated consumer repositories.

## North-Star References

- [Architecture North Star](north_star.md)
- [Technology Stack Baseline](tech_stack.md)

## How To Use

- Start from `docs/platform/architecture/decisions/ADR-0000-template.md`.
- Create decision files as `ADR-<YYYYMMDD>-<slug>.md`.
- Link each implementation-ready work-item spec (`SPEC_READY=true`) to its ADR path.

## Required ADR Coverage

- Business objective and desired timeline
- Functional and non-functional requirement summary
- Selected option + rationale
- Rejected options + rationale
- Affected capabilities/components
- Architecture diagram (Mermaid)
- High-level timeline/work packages (Mermaid Gantt)
- External dependencies
