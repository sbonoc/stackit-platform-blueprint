# Hardening Review

## Repository-Wide Findings Fixed
- Finding 1: `parse_literal_pairs()` silently truncated values at commas — any credential whose value contained a comma (data URIs, base64 payloads with `=` padding, connection strings) was split at the first comma, producing an invalid or missing second field. With `RUNTIME_CREDENTIALS_REQUIRED=false` the reconcile exited 0 without creating the source secret, causing a silent ESO failure cascade that broke app deployments. Fixed by replacing the `IFS=',' read -r -a raw_pairs` loop with a `while IFS= read -r pair` newline-only loop and adding `log_warn` on every parse failure path.

## Observability and Diagnostics Changes
- Metrics/logging/tracing updates: `log_warn` now emitted on every `parse_literal_pairs()` failure (missing `=`, empty key, empty value, detected comma-separated input). Previously only `record_reconcile_issue` was called; with `RUNTIME_CREDENTIALS_REQUIRED=false` the warn was invisible. Now consumers receive a visible stderr diagnostic regardless of the `REQUIRED` setting. Existing `record_reconcile_issue` call preserved — reconcile-issue JSON artifact continues to record the failure.
- Operational diagnostics updates: `record_reconcile_issue` error message updated to reference `key=value` newline-separated as the sole accepted format. `docs/platform/consumer/troubleshooting.md` and its bootstrap template copy updated — seed example at line 528 migrated from comma-separated to `$'...'` quoting form with a link to the migration guide.

## Architecture and Code Quality Compliance
- SOLID / Clean Architecture / Clean Code / DDD checks: Single responsibility preserved — `parse_literal_pairs()` is the only parsing locus. No new abstraction layer introduced. Comma-detection heuristic is self-contained within the function; the heuristic only fires on single-line input where comma-split produces >1 element with an identifier-shaped key and non-empty value in elements [1:], avoiding false positives on base64 padding (`=` trailing produces empty value → not flagged).

One additional regression fixed (line 290: `test_required_mode_does_not_false_flag_hyphenated_target_keys_as_missing` updated). Total: 12/12 tests in `tests/infra/test_runtime_credentials_eso.py` pass. `infra-contract-test-fast`: 136/136 pass. `docs-build` + `docs-smoke`: PASS.
