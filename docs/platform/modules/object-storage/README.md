## Stack Execution Model
- Optional module Make targets are materialized by `make blueprint-render-makefile` (or `make blueprint-bootstrap`) when `OBJECT_STORAGE_ENABLED=true`.
- Scaffolding paths are materialized by `make infra-bootstrap` only when `OBJECT_STORAGE_ENABLED=true`.
- `stackit-*` profiles: managed by Terraform `foundation` layer (`infra/cloud/stackit/terraform/foundation`) with `OBJECT_STORAGE_ENABLED` contract flag.
  - `OBJECT_STORAGE_BUCKET_NAME` is passed through as the canonical bucket contract.
  - Access keys are provider-generated and resolved into runtime artifacts from foundation outputs after apply.
- `local-*` profiles: Helm chart (`bitnami/minio`) using `infra/local/helm/object-storage/values.yaml`.
  - The local fallback uses explicit pinned `docker.io/bitnamilegacy/*` images instead of relying on drifting chart defaults; that registry name is a vendor namespace quirk, while the pinned tags still track the latest stable supported multi-arch line we validate in this repo.
  - Because those pinned legacy images are outside Bitnami's default registry allowlist, the values contract explicitly enables `global.security.allowInsecureImages=true` for this local-only fallback path.
- The pinned legacy images are multi-arch, so the same fallback version contract runs on amd64 CI nodes and arm64 Docker Desktop clusters.
  - STACKIT Object Storage does not expose a provider-side runtime version line in the contract today, so the local MinIO fallback tracks the latest stable compatible chart release rather than a managed-service version number.

## Credentials

**Local lane:** MinIO credentials (`OBJECT_STORAGE_ACCESS_KEY`, `OBJECT_STORAGE_SECRET_KEY`) are reconciled into a Kubernetes Secret `blueprint-object-storage-auth` (keys: `root-user`, `root-password`) on every `infra-object-storage-apply` call via `object_storage_reconcile_runtime_secret`. The Helm values reference this Secret via `auth.existingSecret` — no credentials appear in the rendered values artifact (`artifacts/infra/rendered/object-storage.values.yaml`).

**STACKIT lane:** Credentials are provider-generated by `stackit_objectstorage_credential` and resolved from foundation Terraform outputs after apply. They are never embedded in any checked-in file.

## Smoke Checks

`infra-object-storage-smoke` validates:
- Runtime state file `artifacts/infra/object_storage_runtime.env` exists
- `endpoint` matches `^https?://`
- `bucket` key is present and non-empty
- `region` key is present

On success writes `artifacts/infra/object_storage_smoke.env`.

## Local Lane — STACKIT Lane Differences

| Aspect | Local (`local-*`) | STACKIT (`stackit-*`) |
|---|---|---|
| Provisioning | Bitnami MinIO Helm chart `17.0.21` | Foundation Terraform inline resources |
| Execution class | `fallback_runtime` | `provider_backed` |
| Credentials | Reconciled into K8s Secret `blueprint-object-storage-auth` | Provider-generated via `stackit_objectstorage_credential` |
| Endpoint | `http://<release>.data.svc.cluster.local:9000` | `https://object-storage.<region>.onstackit.cloud` |

## Destroy

```bash
OBJECT_STORAGE_ENABLED=true make infra-object-storage-destroy
```

- **Local lane**: Runs `helm uninstall --ignore-not-found`, then deletes Kubernetes Secret `blueprint-object-storage-auth` (idempotent — tolerates Secret-not-found). Removes all `object_storage_*` state files.
- **STACKIT lane**: Routes to `foundation_reconcile_apply` destroy path. Foundation Terraform destroys the bucket, credentials group, and credential resources.

## Standalone STACKIT Terraform Module

`infra/cloud/stackit/terraform/modules/object-storage/` provides a standalone Terraform module for consumers who want to provision STACKIT object storage outside the foundation deployment pattern. The module mirrors the foundation inline resources and exposes the same 5-key output contract:

```hcl
module "object_storage" {
  source                 = "./modules/object-storage"
  stackit_project_id     = var.stackit_project_id
  stackit_region         = "eu01"
  bucket_name            = "my-bucket"
  credentials_group_name = "my-bucket-creds"
}
# outputs: bucket_name, endpoint_url, access_key, secret_access_key, region
```

The foundation inline resources are not modified — this module is additive with no Terraform state migration required.
