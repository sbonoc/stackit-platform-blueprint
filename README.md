# STACKIT Platform Blueprint (GitHub Template)

This repository is a GitHub template to bootstrap contract-first platform repositories for local and STACKIT environments.

## What This Blueprint Gives You
- Deterministic day-0 onboarding (`blueprint-init-repo`, `blueprint-bootstrap`, `infra-bootstrap`).
- Contract-driven validation and guardrails (`blueprint/contract.yaml` + `make infra-validate`).
- Clear ownership split:
  - blueprint-managed: `make/blueprint.generated.mk`, `scripts/bin/blueprint/**`, `scripts/templates/blueprint/bootstrap/**`
  - platform-owned: `make/platform.mk`, `make/platform/*.mk`, `scripts/bin/platform/**`, `scripts/lib/platform/**`
- Optional modules that are lean-by-default and materialized only when enabled.

## Who Should Use It
- Platform teams starting a new repo with predictable structure, tooling wrappers, and governance.
- Teams that want local-first development with a clean path to STACKIT deployment.

## 5-Minute Quickstart
1. Create your repository from GitHub **Use this template**, then clone it.
2. Initialize repository identity (choose one):
   ```bash
   make blueprint-init-repo-interactive
   ```
   or
   ```bash
   set -a
   source blueprint/repo.init.example.env
   set +a
   make blueprint-init-repo
   ```
3. Bootstrap and validate:
   ```bash
   make blueprint-bootstrap
   make infra-bootstrap
   make infra-validate
   ```
4. Optional smoke:
   ```bash
   make infra-smoke
   ```

Expected outcome:
- `make infra-validate` passes.
- `make help` lists only targets relevant to enabled optional modules.
- `make/blueprint.generated.mk` is rendered and up to date.

## How Repository Generation Works
You do not run an external generator tool.
1. GitHub creates a full copy of this repository when you click **Use this template**.
2. `make blueprint-init-repo` personalizes repo identity values (contract metadata + docs config).
3. `make blueprint-bootstrap` and `make infra-bootstrap` reconcile generated assets and optional scaffolding.

## What To Edit vs Leave Alone
| Area | Edit Policy |
|---|---|
| `make/platform.mk`, `make/platform/*.mk` | Safe to edit for platform-specific targets |
| `scripts/bin/platform/**`, `scripts/lib/platform/**` | Safe to extend for implementation logic |
| `docs/platform/**` | Seeded baseline, then consumer-editable |
| `make/blueprint.generated.mk` | Generated by template, do not edit manually |
| `scripts/bin/blueprint/**`, `scripts/templates/blueprint/bootstrap/**` | Blueprint-managed template surfaces |
| `blueprint/contract.yaml` | Canonical contract; change intentionally and keep docs/tests in sync |

## Common Flows
- Discover commands: `make help`, `make infra-help-reference`
- Enable optional module targets:
  ```bash
  WORKFLOWS_ENABLED=true make blueprint-render-makefile
  WORKFLOWS_ENABLED=true make infra-bootstrap
  WORKFLOWS_ENABLED=true make infra-validate
  ```
- Provision+deploy chain:
  ```bash
  make infra-provision-deploy
  make infra-smoke
  ```
- Upgrade blueprint repo shape:
  ```bash
  make blueprint-migrate
  ```
- Clean generated runtime/build artifacts:
  ```bash
  make blueprint-clean-generated
  ```
